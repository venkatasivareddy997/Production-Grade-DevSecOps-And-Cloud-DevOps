# Module 8 – Nexus Artifact Management

Artifact management is a critical part of the CI/CD lifecycle.

After applications are built, the generated artifacts must be stored, versioned, and shared reliably across teams and environments. This is where artifact repositories like Nexus are used.

## This module focuses on:

- Understanding artifact management in DevOps  
- Setting up and configuring Nexus Repository Manager  
- Storing and managing build artifacts  
- Publishing artifacts from different tech stacks  
- Using Nexus in CI/CD pipelines  
- Working with Docker registries in Nexus  
- Managing artifact versions and access  

This is a hands-on, repository-focused module aligned with real industry practices.

## What You Will Learn

- What artifact management is and why it is important  
- Types of repositories (hosted, proxy, group)  
- Nexus architecture and components  
- Publishing Node.js and Java artifacts to Nexus  
- Downloading and consuming artifacts in pipelines  
- Versioning strategies for artifacts  
- Using Nexus as a private Docker registry  
- Integrating Nexus with CI/CD pipelines  
- Managing access and credentials securely  

## How to Complete This Module

- Complete all tasks listed in tasks.md  
- Install and configure Nexus locally or on a VM  
- Publish artifacts from real projects  
- Download and use artifacts in pipelines  
- Configure Docker registry in Nexus  
- Verify artifact storage and versioning  
- Document configurations and workflows  
- Push your work to your repository  
- Share your learning publicly  

Artifacts are central to deployments.  
Understanding how they are stored and used is essential for DevOps engineers.

## Outcome of This Module

After completing this module, you will be able to:

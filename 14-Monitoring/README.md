# Module 14 – Monitoring

Monitoring and observability are critical for maintaining reliable systems in production.

In real-world environments, DevOps engineers must track application health, infrastructure performance, logs, and alerts to ensure systems remain stable and issues are detected early.

This module focuses on:

- Setting up monitoring using Prometheus and Grafana
- Monitoring infrastructure and applications
- Configuring alerting systems using Alertmanager
- Implementing blackbox and node-level monitoring
- Monitoring containerized environments using Docker
- Monitoring Kubernetes clusters including EKS
- Centralized logging using ELK stack
- Event streaming and log pipelines using Kafka

This is a hands-on, production-oriented module focused on real observability setups.

## What You Will Learn

- Monitoring fundamentals and why observability matters
- Prometheus architecture and metrics collection
- Grafana dashboards and visualization
- Node Exporter for system metrics
- Blackbox Exporter for endpoint monitoring
- Alertmanager for email alerts and notifications
- Monitoring Docker containers
- Kubernetes monitoring using Prometheus stack
- Monitoring EKS clusters
- Centralized logging using ELK (Elasticsearch, Logstash, Kibana)
- Log ingestion pipelines using Kafka
- Troubleshooting monitoring and alerting issues

- Configure exporters and dashboards
- Implement alerting mechanisms
